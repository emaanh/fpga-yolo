# tiny_yolo: streaming-video CNN object detector on FPGA

A Tiny-YOLO-style object detector implemented in synthesizable Verilog.
Three conv blocks + a 1×1 detection head classify and localize shapes on
a 32×32 RGB frame, producing a 4×4 grid of bounding-box predictions per
frame.

**Authors:** Emaan Heidari ‹eheidari@usc.edu›, Abir Bhatt ‹abirbhat@usc.edu›

## Architecture summary

| stage   | shape in     | shape out    | RTL module           | cycles    |
|---------|--------------|--------------|----------------------|----------:|
| load    | external     | 32×32×3      | (host)               |    ~3,000 |
| conv1   | 32×32×3      | 32×32×8      | `conv3x3` (3→8)      |    ~238 K |
| pool1   | 32×32×8      | 16×16×8      | `max_pool2x2`        |       8 K |
| conv2   | 16×16×8      | 16×16×16     | `conv3x3` (8→16)     |    ~303 K |
| pool2   | 16×16×16     | 8×8×16       | `max_pool2x2`        |       4 K |
| conv3   | 8×8×16       | 8×8×32       | `conv3x3` (16→32)    |    ~299 K |
| pool3   | 8×8×32       | 4×4×32       | `max_pool2x2`        |       2 K |
| head    | 4×4×32       | 4×4×7        | `conv1x1` (32→7)     |     ~4 K  |
| sigmoid | 4×4×7        | 4×4×7        | `sigmoid_pass` + LUT |   112     |

Total ≈ 858 K cycles per frame; at 100 MHz that's 8.6 ms ≈ 116 fps.

## Fixed-point format

All activations and weights are Q4.12 signed (1 sign + 3 int + 12 frac,
range ≈ [−8, +8)). Internal MAC accumulators are 32 bits (Q8.24). The
final write back to Q4.12 goes through `saturate.v`, so wraparound never
silently corrupts the output.

LeakyReLU uses slope = 1/8 so the negative branch is a single
arithmetic shift (`leaky_relu.v`).

Sigmoid is a 1024-entry combinational LUT generated by
`python/gen_sigmoid_lut.py`. Index = top 10 bits of (input with sign
flipped), which maps the Q4.12 range [−8, +8) onto the LUT evenly.

## Memory map

Two ping-pong feature maps (`fmap_mem.v`, distributed-RAM style):

```
load (ext)  -> A
conv1: A->B   pool1: B->A
conv2: A->B   pool2: B->A
conv3: A->B   pool3: B->A
head:  A->B   sigmoid: B->A
ext read of A holds the final sigmoided cells (112 words)
```

Each fmap stores a CHW-planar tensor: address = `c * H * W + y * W + x`.

Weight ROMs are layer-private, initialised from `.mif` files via
`$readmemh` in `weight_rom.v`. Weight layout (OIHW flattened):
`addr = co * Cin * 9 + ci * 9 + ky * 3 + kx` for 3×3,
`addr = co * Cin + ci` for the 1×1 head.

## Why distributed RAM, not BRAM?

`fmap_mem` and `weight_rom` use *combinational* read so the conv FSM can
do one MAC per cycle. Swapping to BRAM is a one-cycle pipeline register
on `rdata` and a single bubble in the address counter. The MAC datapath
itself doesn't change.

## Build & run

Requires Python 3.9+, numpy, pillow. Optional: PyTorch for training.

```bash
# from tiny_yolo/
./scripts/run_sim.sh                   # one frame, random weights (smoke test)
./scripts/run_sim.sh -f 3              # frame_0003
./scripts/run_sim.sh --regen-data 2000 # regenerate 2000 synthetic frames
./scripts/run_sim.sh --trained         # quantize data/model.pt and run
./scripts/run_sim.sh --vcd             # also dump build/tb_top.vcd
```

Outputs land in:
- `build/sim_frame_XXXX.log`   — cycle-by-cycle stage transitions
- `data/detections/cells_XXXX.txt` — 112 raw Q4.12 hex values from the head
- `data/detections/frame_XXXX.txt` — `<cls> <cx> <cy> <w> <h> <conf>` after NMS
- `data/detections/frame_XXXX_vis.png` — input frame with boxes drawn

## Training a real model

The default `random_weights.py` makes the pipeline run but the
predictions are noise. To train:

```bash
pip install torch
python python/gen_dataset.py -n 2000      # synthetic shapes dataset
python python/train.py --epochs 30        # writes data/model.pt
python python/quantize.py                 # exports Q4.12 .mif weights
./scripts/run_sim.sh --trained
```

`python/model.py` defines the PyTorch model with the exact architecture
the RTL implements; training is YOLOv1-style (coord MSE for cells with a
target, BCE on objectness for all cells with `λ_noobj=0.5`, BCE on
class logits for target cells).

```
tiny_yolo/
├── python/
│   ├── config.py              shared shapes + Q4.12 constants
│   ├── utils.py               .mif / .hex helpers
│   ├── gen_dataset.py         synthesize circles + squares + ground truth
│   ├── gen_sigmoid_lut.py     emit data/sigmoid/sigmoid_lut.mif
│   ├── random_weights.py      smoke-test Q4.12 weights (no training needed)
│   ├── model.py               PyTorch TinyYolo (mirrors the RTL)
│   ├── train.py               train + checkpoint
│   ├── quantize.py            FP32 → Q4.12 weight .mif exporter
│   ├── decode_nms.py          read cells dump → detections + NMS
│   └── visualize.py           overlay boxes on the input frame
├── rtl/
│   ├── include.vh             single source of truth (shapes + Q4.12)
│   ├── fmap_mem.v             feature-map memory (CHW planar)
│   ├── weight_rom.v           $readmemh-initialised weight/bias ROM
│   ├── leaky_relu.v           combinational leaky ReLU (slope 1/8)
│   ├── saturate.v             32-bit Q8.24 → 16-bit Q4.12 with clamp
│   ├── sigmoid_lut.v          1024-entry combinational sigmoid LUT
│   ├── sigmoid_pass.v         walk fmap, rewrite each cell through σ
│   ├── max_pool2x2.v          stride-2 max pool, channel-major FSM
│   ├── conv3x3.v              same-padding 3×3 conv, 1 MAC/cycle FSM
│   ├── conv1x1.v              detection-head 1×1 conv
│   └── tiny_yolo_top.v        chains the layers, A/B ping-pong, stage FSM
├── tb/
│   └── tb_top.v               load frame, run inference, dump cells
├── scripts/
│   └── run_sim.sh             end-to-end demo
└── data/                      generated artifacts (frames, weights, LUTs, etc.)
```

## Porting to a real camera

The interface to swap is `tiny_yolo_top.ext_*` — Q4.12 pixels are
streamed in through that port. Replace the file loader with:

1. A camera-receiver block (e.g. MIPI CSI-2 → debayer → resize to 32×32
   → normalize to Q4.12) driving `ext_we / ext_waddr / ext_wdata`.
2. An HDMI/VGA framebuffer that reads predictions from `ext_rdata`
   after each `done` pulse and overlays boxes via a sprite blender.

Neither the conv pipeline nor the memory layout needs to change.
